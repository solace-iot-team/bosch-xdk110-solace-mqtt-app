◆ appCmdCtrl_PreprocessSubscriptionCallback()

static void appCmdCtrl_PreprocessSubscriptionCallback ( void *  paramsPtr,
uint32_t  processTypeParam 
)
static

Preprocess the subscription callback. Enqueued from appCmdCtrl_SubscriptionCallBack().

Validates the topic contains either 'configuration' or 'command' to determine the instruction type. Parses the incoming message as JSON and sends a failed response if not successful. Extracts the exchangeId from the incoming message (sends failed response if not successful). Extracts optional 'tags' element from the incoming message. Creates the response message with exchangeId, tags. Finally, checks if module is still busy processing a previous instruction and sends a failed reponse. Calls appCmdCtrl_ProcessInstruction() to process the instruction.

Exceptions
Retcode_RaiseErrorRETCODE(RETCODE_SEVERITY_FATAL, RETCODE_SOLAPP_CMD_CTRL_LISTENING_TO_WRONG_TOPICS)

Sets up the subscriptions based on the topic configuration.

Parameters
[in]configPtrthe topic configuration
Returns
Retcode_T: RETCODE_OK
Retcode_T: retcode from AppMqtt_Subscribe()

The following subscription topics are constructed using AppMisc_FormatTopic():

Single device command and configuration topic:

AppMisc_FormatTopic("%s/iot-control/%s/device/%s/command", methodCreate, baseTopic, appCmdCtrl_DeviceId)
AppMisc_FormatTopic("%s/iot-control/%s/device/%s/configuration", methodUpdate, baseTopic, appCmdCtrl_DeviceId)

All devices in the same 3 level resource categorization:

AppMisc_FormatTopic("%s/iot-control/%s/device/command", methodCreate, baseTopic, NULL)
AppMisc_FormatTopic("%s/iot-control/%s/device/configuration", methodUpdate, baseTopic, NULL)

All devices in the same 2 level resource categorization:

AppMisc_FormatTopic("%s/iot-control/%s/%s/device/command", methodCreate, baseTopicLevelsArray[0], baseTopicLevelsArray[1])
AppMisc_FormatTopic("%s/iot-control/%s/%s/device/configuration", methodUpdate, baseTopicLevelsArray[0], baseTopicLevelsArray[1])

All devices in the same 1 level resource categorization:

AppMisc_FormatTopic("%s/iot-control/%s/device/command", methodCreate, baseTopicLevelsArray[0], NULL)
AppMisc_FormatTopic("%s/iot-control/%s/device/configuration", methodUpdate, baseTopicLevelsArray[0], NULL)

◆ appCmdCtrl_SubscriptionCallBack()

static void appCmdCtrl_SubscriptionCallBack ( AppXDK_MQTT_IncomingDataCallbackParam_T *  params)
static

The global subscription callback.

Blocks instruction processing and enqueues appCmdCtrl_PreprocessSubscriptionCallback(). Allows only 1 instruction to be processed at a time, enqueues the 2. instruction. Any more instructions are discarded, providing (some) protection against too many instructions sent in close sequence.

type: AppXDK_MQTT_IncomingDataCallback_Func_T()

Parameters
[in]paramsthe incoming subscription parameters.
Exceptions
Retcode_RaiseErrorRETCODE(RETCODE_SEVERITY_FATAL, RETCODE_SOLAPP_CMD_CTRL_FAILED_TO_ENQUEUE_INSTRUCTION_PROCESSING)
