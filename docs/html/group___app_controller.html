Detailed Description

Solace XDK110 application controller.

Application controller for the Solace App.

Controls the initialization, setup and enable functions of all modules.

Controls the overall flow of the app: excuting commands and configuration requests, telemetry sampling and publishing, re-connects to the broker, etc.

Author
Solace IoT Team
Date
Jan-2019 to Feb-2020

◆ appController_SetupAfterDisconnect()

static void appController_SetupAfterDisconnect ( void *  param1,
uint32_t  param2 
)
static

Setup after a disconnect event from the broker.

Enqueued by appController_MqttBrokerDisconnectCallback(), runs in AppController command processor. Waits until AppController is finished processing any other instructions and then blocks new instruction processing until finished.

Sequence:

  • notifies modules of the disconnect event and suspends telemetry tasks.
  • reconnect:
    • checks if WLAN connection still active, waits APP_CONTROLLER_WLAN_RECONNECT_WAIT_MS milliseconds for APP_CONTROLLER_WLAN_RECONNECT_MAX_TRIES times until it is
    • connects to the broker APP_CONTROLLER_MQTT_RECONNECT_MAX_TRIES times with a wait of APP_CONTROLLER_MQTT_RECONNECT_WAIT_MS before trying again
    • if it can't connect, raises a FATAL error
  • notifies modules of the reconnect and starts telemetry tasks again.
Parameters
[in]param1unused
[in]param2unused
Exceptions
Retcode_RaiseError(RETCODE_SEVERITY_FATAL) if it can't get an internet connection / connect to the broker
Retcode_RaiseError- with called functions' retcode if not RETCODE_OK
Note
It can happen that after a reconnect another disconnect event occurs. Hence, all suspend and setup functions must handle being called again.
