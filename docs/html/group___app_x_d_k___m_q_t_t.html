Detailed Description

Implements the MQTT interface to the ServalPal module. Adapted from the original XDK module: XDK_MQTT.h and MQTT.c.

Added single use / single call protection using semaphores. Added internal state management.

Author
Solace IoT Team
Date
Jan-2019 to Feb-2020

◆ AppXDK_MQTT_ConnectToBroker()

Retcode_T AppXDK_MQTT_ConnectToBroker ( const AppXDK_MQTT_Connect_T *  connectPtr)

Connect to the mqtt broker.

Blocks module using semaphore appXDK_MQTT_ExternalInterface_SemaphoreHandle and sets appXDK_MQTT_State to AppXDK_MQTT_State_Connecting and to AppXDK_MQTT_State_Ready when finished.

Uses appXDK_MQTT_ConnectSemaphoreHandle for synchronization with appXDK_MQTT_EventHandler().

Note
Only supports AppXDK_MQTT_TypeServalStack.
Parameters
[in]connectPtrthe connect information
Returns
Retcode_T: RETCODE_OK
Retcode_T: RETCODE(RETCODE_SEVERITY_FATAL, RETCODE_SOLAPP_APP_XDK_MQTT_UNSUPPORTED_SCHEME)
Retcode_T: RETCODE(RETCODE_SEVERITY_ERROR, RETCODE_SOLAPP_APP_XDK_MQTT_URL_PARSING_FAILED)
Retcode_T: RETCODE(RETCODE_SEVERITY_ERROR, RETCODE_SOLAPP_APP_XDK_MQTT_SERVAL_MQTT_CONNECT_CALL_FAILED) when the call to Mqtt_connect() itself fails
Retcode_T: RETCODE(RETCODE_SEVERITY_FATAL, RETCODE_SOLAPP_APP_XDK_MQTT_TIMEOUT_NO_CONNECT_CALLBACK_RECEIVED_FROM_BROKER) if event handler does not receive any callback in time.
Retcode_T: RETCODE(RETCODE_SEVERITY_FATAL, RETCODE_SOLAPP_APP_XDK_MQTT_CALLBACK_FAILED) if appXDK_MQTT_EventHandler_Retcode not RETCODE_OK
Retcode_T: RETCODE(RETCODE_SEVERITY_ERROR, RETCODE_SOLAPP_APP_XDK_MQTT_CONNECTION_ERROR_RECEIVED_FROM_BROKER) if connection to broker failed

◆ appXDK_MQTT_EventHandler()

static retcode_t appXDK_MQTT_EventHandler ( MqttSession_T *  servalSession,
MqttEvent_t  servalEvent,
const MqttEventData_t *  servalEventData 
)
static

Callback function used by the stack to communicate events to the application. Each event will bring with it specialized data that will contain more information.

Module ensures there is only one callback running at any time. The initiator of the interaction - AppXDK_MQTT_ConnectToBroker(), AppXDK_MQTT_SubsribeToTopic(), AppXDK_MQTT_UnsubsribeFromTopics(), AppXDK_MQTT_PublishToTopic() will block on (take) a semaphore. This callback event handler will release the semaphore so the initiator can continue synchronously. Implements a typical async -> sync conversion using semaphores.

Sets the appXDK_MQTT_EventHandler_Retcode variable for the initiator of the interaction to read:

  • appXDK_MQTT_EventHandler_Retcode: RETCODE(RETCODE_SEVERITY_ERROR, RETCODE_SOLAPP_APP_XDK_MQTT_CONNECT_SEMAPHORE_ERROR)
  • appXDK_MQTT_EventHandler_Retcode: RETCODE(RETCODE_SEVERITY_ERROR, RETCODE_SOLAPP_APP_XDK_MQTT_SUBSCRIBE_SEMAPHORE_ERROR)
  • appXDK_MQTT_EventHandler_Retcode: RETCODE(RETCODE_SEVERITY_ERROR, RETCODE_SOLAPP_APP_XDK_MQTT_UNSUBSCRIBE_SEMAPHORE_ERROR)
  • appXDK_MQTT_EventHandler_Retcode: RETCODE(RETCODE_SEVERITY_ERROR, RETCODE_SOLAPP_APP_XDK_MQTT_PUBLISH_SEMAPHORE_ERROR)
  • appXDK_MQTT_EventHandler_Retcode: RETCODE(RETCODE_SEVERITY_ERROR, RETCODE_SOLAPP_APP_XDK_MQTT_UNHANDLED_MQTT_EVENT)
  • appXDK_MQTT_EventHandler_Retcode: RETCODE(RETCODE_SEVERITY_ERROR, RETCODE_SOLAPP_APP_XDK_MQTT_UNKNOWN_CALLBACK_EVENT_RECEIVED)

Sets the following status variables for the initiator of the interaction to read:

  • appXDK_MQTT_ConnectionStatus: true for ok, false for failed
  • appXDK_MQTT_SubscriptionStatus: true for ok, false for failed
  • appXDK_MQTT_UnsubscribeStatus: true for ok, false for failed
  • appXDK_MQTT_PublishStatus: true for ok, false for failed

Manages / sets the following static variables:

  • appXDK_MQTT_EventHandler_ServalEvent
  • appXDK_MQTT_EventHandler_PriorServalEvent
Parameters
[in]servalSessionunused
[in]servalEventevent type
[in]servalEventDatathe data associated for this event type, see MqttEventData_u
Returns
retcode_t: always RC_OK
union MqttEventData_u {
MqttConnectionEstablishedEvent_T connect;
MqttSubscriptionData_T subscription;
MqttPublishData_T publish;
MqttSession_T session;
retcode_t status;
} MqttEventData_t;

Events communicated to the application using the onMqttEvent callback in the session. Each event carries a particular data type which describes further the event.

The mapping of the MqttEvent_t type field to MqttEventData_t fields is:

  • MQTT_CONNECTION_ESTABLISHED -> MqttConnectionEstablishedEvent_T,
  • MQTT_CONNECTION_ERROR -> MqttConnectionEstablishedEvent_T,
  • MQTT_CONNECT_SEND_FAILED -> NULL,
  • MQTT_CONNECT_TIMEOUT -> NULL,
  • MQTT_CONNECTION_CLOSED -> NULL,
  • MQTT_DISCONNECT_SEND_FAILED -> NULL,
  • MQTT_PING_SEND_FAILED -> NULL,
  • MQTT_PING_RESPONSE_RECEIVED -> NULL,
  • MQTT_SUBSCRIPTION_ACKNOWLEDGED -> MqttSubscriptionData_T,
  • MQTT_SUBSCRIBE_SEND_FAILED -> MqttSubscriptionData_T,
  • MQTT_SUBSCRIBE_TIMEOUT -> MqttSubscriptionData_T,
  • MQTT_SUBSCRIPTION_REMOVED -> MqttSubscriptionData_T,
  • MQTT_UNSUBSCRIBE_TIMEOUT -> MqttSubscriptionData_T,
  • MQTT_UNSUBSCRIBE_SEND_FAILED -> MqttSubscriptionData_T,
  • MQTT_INCOMING_PUBLISH -> MqttPublishData_T,
  • MQTT_PUBLISHED_DATA -> MqttPublishData_T,
  • MQTT_PUBLISH_TIMEOUT -> MqttPublishData_T,
  • MQTT_PUBLISH_SEND_FAILED -> MqttPublishData_T,
  • MQTT_PUBLISH_SEND_ACK_FAILED -> NULL,
  • MQTT_SERVER_DID_NOT_RELEASE -> NULL,
